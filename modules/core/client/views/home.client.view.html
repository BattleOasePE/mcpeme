Get started

The old version of mcpe.me has been decommissioned and replaced with this website. You will need to create an account and sign in. Then you can click "Records" and "Create records" in the menu bar. Enter your details and your record will be created instantly. What you put in "content" depends on your record type, but it should be some way to connect to your server.
  • CNAME should be your existing hostname (eg: foo.example.com)
  • A should be your server IPv4 address (eg: 1.2.3.4)
  • AAAA should be your server IPv6 address (eg: 2001:0db8:0000:0042:0000:8a2e:0370:7334)

These services cost money, but that is only logical, running a server does cost money. Verification is a process which ensures that the record owner is also the server owner. Only MCPE servers can be verified. You may be requested to verify your server by our staff for security reasons.

You can verify a server by setting the title of the server to the "Record ID" and restarting the server. mcpe.me will verify you server a few seconds after you add the server. If your server is confirmed to be an MCPE server (but doesn't have the verification code set), mcpe.me will try again in 20 minutes. You can force mcpe.me to re-verify by submitting the edit form without any changes. Verification is not instant, it will take about 15 second for mcpe.me to complete a verification.   • User ID - this is a unique identifier for a user that can't be changed. It is useful for reporting users.
  • Record ID - this is a unique identifier for a record that can't be changed. It is useful for requesting support with records.
  • CF ID - each record is assigned an ID by CloudFlare. This ID may change over the course of the record's life and is only used internally.
  • Record API key - each record has a key generated for performing remote updates on it. This key is not yet used.
